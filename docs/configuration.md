# Configuration

## Environment Variables

All variables are loaded and validated by `src/shared/loadEnv.ts` using Zod schemas. The loader walks up to 4 directory levels looking for `.env.local` or `.env`, and validates the full config on first access.

### Core

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `NODE_ENV` | string | `development` | Standard Node.js environment |
| `ZEPHYR_ENV` | enum | `local` | Network selection: `local`, `sepolia`, `mainnet` |

### Database

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `DATABASE_URL` | string | -- | PostgreSQL connection URL (`postgresql://...`) |

### Zephyr Chain

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `ZEPHYR_D_RPC_URL` | URL | `http://remote-node.zephyrprotocol.com:17767` | Zephyr daemon RPC |
| `ZEPHYR_WALLET_RPC_URL` | URL | -- | Wallet-engine RPC for mints/redeems/transfers |
| `ZEPHYR_WALLET_PASSWORD` | string | -- | Wallet unlock password |
| `ZEPHYR_BRIDGE_ADDRESS` | string | -- | Bridge contract address on Zephyr chain |
| `BRIDGE_API_URL` | URL | -- | Bridge REST API endpoint |
| `ZEPHYR_PAPER` | boolean | `false` | Simulate native Zephyr operations |

### EVM RPC

RPC URLs are network-specific. The loader resolves `RPC_URL_HTTP` and `RPC_URL_WS` from the `ZEPHYR_ENV` value:

| Variable | Network | Description |
|----------|---------|-------------|
| `RPC_URL_LOCAL_HTTP` | local | HTTP RPC (default `http://127.0.0.1:8545`) |
| `RPC_URL_LOCAL_WS` | local | WebSocket RPC (default `ws://127.0.0.1:8545`) |
| `RPC_URL_SEPOLIA_HTTP` | sepolia | HTTP RPC |
| `RPC_URL_SEPOLIA_WS` | sepolia | WebSocket RPC |
| `RPC_URL_MAINNET_HTTP` | mainnet | HTTP RPC |
| `RPC_URL_MAINNET_WS` | mainnet | WebSocket RPC |

Legacy fallback variables (`RPC_URL_LOCAL`, `RPC_URL_SEPOLIA`, `RPC_URL_MAINNET`) are supported but deprecated.

**Resolution logic:** `env.RPC_URL_HTTP` returns the HTTP URL for the current network. `env.RPC_URL_WS` returns the WebSocket URL, falling back to converting the HTTP URL (`http://` to `ws://`, `https://` to `wss://`).

## Address Configuration

Network-specific address files live at `src/services/evm/config/addresses.{network}.json`. On devnet, this file is generated by [bridge-orchestration](https://github.com/fennzar/bridge-orchestration) and synced via `make sync-env`.

### Structure

```json
{
  "version": 1,
  "chainId": 31337,
  "contracts": {
    "poolManager": "0x...",
    "positionManager": "0x...",
    "stateView": "0x...",
    "v4Quoter": "0x...",
    "swapRouter": "0x...",
    "permit2": "0x..."
  },
  "tokens": {
    "USDT": { "address": "0x...", "decimals": 6, "symbol": "USDT", "name": "Tether USD" },
    "wZEPH": { "address": "0x...", "decimals": 12, "symbol": "wZEPH", "name": "Wrapped Zephyr" }
  },
  "pools": [ ... ],
  "seeding": { ... }
}
```

### Contracts (per network)

| Contract | Purpose |
|----------|---------|
| `poolManager` | Uniswap V4 PoolManager -- central pool registry |
| `positionManager` | Uniswap V4 PositionManager -- LP position NFTs |
| `stateView` | On-chain pool state reader |
| `v4Quoter` | On-chain swap quote simulator |
| `swapRouter` | Universal swap router |
| `permit2` | Uniswap Permit2 -- gasless token approvals |

### Tokens

Six tokens per network. Wrapped Zephyr assets use 12 decimals; USDT/USDC use 6.

| Symbol | Decimals | Description |
|--------|----------|-------------|
| USDC | 6 | USD Coin |

| wZSD | 12 | Wrapped Zephyr Stable Dollar |

## Prisma Data Model

Schema at `src/infra/prisma/schema.prisma`. Use the `scripts/prisma-with-env.sh` wrapper to run Prisma commands (it loads `.env` first).

### EVM/Pool Models

| Model | Purpose |
|-------|---------|
| `Token` | EVM token registry (address, symbol, decimals per chain) |
| `Pool` | Liquidity pool metadata (protocol, tokens, fee tier, tick spacing) |
| `PoolStateSnapshot` | Historical pool state: liquidity, sqrtPriceX96, tick, reserves, TVL |
| `ScanCursor` | Watcher indexing progress -- last scanned block per task |
| `PoolDiscoveryEvent` | Audit log of pool creation events |
| `SwapEvent` | Historical swap transactions with amounts, price, tick |
| `Position` | LP positions: owner, tick range, liquidity, amounts, fees |
| `PositionSnapshot` | Historical position state at specific blocks |

### Engine Models

| Model | Purpose |
|-------|---------|
| `MarketSnapshot` | MEXC market data: bid/ask prices, order book depth |
| `ReserveSnapshot` | Zephyr reserve state: RR, ZEPH price, supply figures, policy flags |
| `OperationQueue` | Manual approval workflow: pending/approved/executing/completed/failed |
| `ExecutionHistory` | Audit trail: plan, result, step results, PnL, gas used |
| `InventoryBalance` | Current balances per asset per venue |
| `EngineSettings` | Singleton runtime config: autoExecute, manualApproval, cooldownMs |
| `LPPosition` | App-level LP position tracking with target mode and rebalance history |

### Decimal Precision Conventions

| Prisma Type | Used For | Range |
|-------------|----------|-------|
| `Decimal(65, 30)` | On-chain values: liquidity, sqrtPriceX96, reserves, swap amounts | Full EVM uint256 range |
| `Decimal(30, 12)` | Token balances, quantities, inventory amounts | Matches Zephyr 12-decimal atomic units |
| `Decimal(20, 8)` | USD prices, PnL values | Sub-cent precision |
| `Decimal(30, 0)` | Gas used (integer wei) | Integer-only |
| `Decimal(10, 4)` | Reserve ratio percentages | e.g., 400.0000 = 400% |

Native Zephyr amounts in the database are stored in atomic units. Divide by 1e12 for human-readable values.
